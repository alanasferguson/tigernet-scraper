1. Problem Decomposition: How did you break this problem down? What were the sub-problems you identified?
    - Sub-Problem 1: Authentication. TigetNet, like all Princeton apps, sits behind CAS and Duo 2-Factor Authentication. I couldn't make a request and get data back, and brainstormed how I could programmatically log in and maintain a valid session across multiple requests. I worked on translating encoding my own session cookies adn tokens into the hidden .env file into providing my username and password for the program to use as input for automating the process. This worked but realistically was not a proper solution and compromised my own personal data. I did research on playwright and rebuilt the pipeline to automate the login flow, including dismissing the cookie consent popup and navigating the CAS login form. There is no "safe" or. "ethical" way to get around the 2-Factor Authentication, which is the one remaining human step. Taking Information Security and Systems this semester was especially helpful in breaking down how to approach most feasibly automating this process without flagging my account. 
    - Sub-Problem 2: Data Access. I manually needed to look through how TigerNet stores and serves its data. I used DevTools initially in chrome and looked at network requests that popped up as I looked at both individual profiles and also the alumni directory. I discovered that Tigernet was built on Hiverbite and saw two JSON API endpoints: /frontoffice/api/users which was a paginated directory listeing and a profile endpoint at /private/frontoffice/users/profiles/{id}. 
    - Sub-Problem 3: CSV Output Reliability and Readibility: After realizing that each user's data was coming from two separate API calls in large JSON files, I neeed to figure out how to take the responses and merge them into two different fields. I eventually used a flattening function to explore this. 
    - Network Timeout: Given that there are over 130,000 profiles on Tigernet, running the entire file takes a lot of time. I built a retry logic that also incorporate a progress checkpoint to save after every completed page so that restarts pick up where they left off.
